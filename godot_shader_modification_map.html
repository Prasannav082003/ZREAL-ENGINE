Shader & source modification guide — by quality goal

Highest impactForward+

Main PBR + Lighting Shader

servers/rendering/renderer_rd/shaders/forward_clustered/scene_forward_clustered.glsl

Shadow PCF quality — search shadow_sample_count or SHADOW_MODE_PCF_5. Increase sample count from 5→13 (Poisson disk) for softer shadows.

PCSS soft shadows — the angular distance blocker search; search penumbra_size. Increase blocker search taps from 8→16.

GGX specular — search D_GGX. Replace with a more accurate GTR2 distribution for sharper highlights at low roughness.

Clearcoat / anisotropy — search CLEARCOAT, ANISOTROPY. Improve the per-lobe Fresnel integration here.

IBL (reflection capture quality) — search textureLod(radiance_map. Increase mip-bias compensation for sharper reflections.

High impact

Tone Mapping

servers/rendering/renderer_rd/shaders/effects/tonemap.glsl

Operator selection — search TONEMAPPER_LINEAR, TONEMAPPER_REINHARDT, TONEMAPPER_FILMIC, TONEMAPPER_ACES. Add AgX by inserting a new branch.

Exposure — search exposure uniform. Auto-exposure response curve is computed above the tone-map call.

Color grading LUT — search color_correction sampler. Increase LUT resolution from 32³ → 64³ for smoother gradients.

White point — search white uniform — controls the shoulder of Filmic/ACES.

Medium impact

SSAO / SSIL

servers/rendering/renderer_rd/shaders/effects/ssao.glsl
servers/rendering/renderer_rd/shaders/effects/ssil.glsl
servers/rendering/renderer_rd/effects/ss_effects.cpp

Sample count — in ss_effects.cpp, search quality enum or ssao_num_samples. Increasing from Medium (8) → Ultra (32) dramatically reduces noise.

Blur passes — search ssao_blur_passes. Adding a 3rd bilateral blur pass removes banding with little cost.

Depth-aware weighting — in ssao.glsl, search depth_fade. Tighten the falloff for contact shadows to appear only on close surfaces.

High impact

Glow / Bloom

servers/rendering/renderer_rd/shaders/effects/glow.glsl
drivers/gles3/effects/glow.cpp (Compatibility)

Blur kernel size — search GLOW_FILTER_GAUSSIAN, GLOW_FILTER_BICUBIC. Bicubic mode gives smoother bloom at 2× cost.

Mip levels — search glow_levels. Using 7 mip levels (vs default 5) adds wider, more physically plausible scatter.

Threshold knee — search glow_bloom, glow_hdr_threshold. A softer knee curve (quadratic instead of hard step) removes the harsh bloom edge.

Medium impact

Screen Space Reflections

servers/rendering/renderer_rd/shaders/effects/ssr.glsl
servers/rendering/renderer_rd/shaders/effects/ssr_filter.glsl

Ray march steps — search max_steps or ssr_max_steps. Increasing from 64→128 prevents rays missing geometry at distance.

Binary refinement iterations — search BINARY_SEARCH_ITERATIONS. More iterations → sharper reflection edges at surfaces.

Roughness fade — search roughness_fade. Widen the fade ramp to allow SSR on slightly rough surfaces.

Medium impact

IBL Prefilter (Reflections)

servers/rendering/renderer_rd/shaders/effects/cubemap_roughness.glsl

Sample count — search NUM_SAMPLES. Increase from 512 → 2048 for smoother IBL at the cost of a slower bake.

Importance sampling — search ImportanceSampleGGX. This can be swapped for a multi-scattering GGX (Karis 2013) to prevent energy loss on rough metals.

No C++ neededGDScript

Custom Compositor Effect

Extend CompositorEffect — no engine rebuild required

Create a @tool GDScript extending CompositorEffect. Hook into EFFECT_CALLBACK_TYPE_POST_TRANSPARENT or POST_SKY. Write a compute shader as a string, compile it with RenderingDevice, and dispatch it in _render_callback(). Gives you full access to color, depth, and normal/roughness buffers without recompiling Godot.

Compatibility renderer (GLES3/OpenGL)

Equivalent files for the Compatibility renderer

drivers/gles3/shaders/scene.glsl — main PBR pass
drivers/gles3/shaders/tonemap_inc.glsl — tone mapping
drivers/gles3/effects/glow.cpp + glow.glsl — bloom
drivers/gles3/effects/post_effects.cpp — DOF, adjustments
drivers/gles3/shaders/effects/post.glsl — final composite

The Compatibility renderer has fewer post-processing stages and no SSAO/SSR, but all lighting and tone mapping changes mirror the same sections as the Forward+ shaders above. Look for the same uniform and function names — they are intentionally kept in sync.
